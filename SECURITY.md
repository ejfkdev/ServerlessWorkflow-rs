# Security Policy

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability in ServerlessWorkflow-rs, please report it responsibly.

**Do not file a public issue.** Instead, please email:

[cncf.serverless.workflow@gmail.com](mailto:cncf.serverless.workflow@gmail.com)

Include the following information:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to provide a detailed response within 7 days.

## Scope

This policy covers:

- The ServerlessWorkflow-rs source code in this repository
- The `swf` CLI binary distributed via GitHub Releases
- Dependencies with known vulnerabilities

Out of scope:

- Vulnerabilities in third-party services
- Social engineering attacks
- Denial of service attacks against public infrastructure

## Disclosure Policy

We follow coordinated disclosure:

1. Report is received and acknowledged
2. Maintainers investigate and confirm the vulnerability
3. A fix is developed and a security advisory is drafted
4. The fix is released and the advisory is published

We credit reporters in our security advisories unless they request anonymity.

## Supported Versions

| Version | Supported |
|---------|-----------|
| `1.0.0-alpha` | Yes |
| Prior pre-releases | No |
